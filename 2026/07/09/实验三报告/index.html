实验三报告

实验三 代码编辑器 —— 实验报告

一、实验目的

本次实验旨在安装并使用 Visual Studio Code 代码编辑器，通过安装 Markdown 相关插件掌握编辑器插件的扩展机制，并运用 Markdown 基础语法完成一个文档的编辑与实时渲染。

二、实验环境

  • 操作系统：Windows 11 家庭中文版（内部版本号 10.0.26100）
  • 代码编辑器：Visual Studio Code
  • 核心插件：Markdown All in One 3.6.3、Markdown Preview Enhanced 0.8.30
  • 辅助插件：Prettier（格式化）、markdownlint（语法检查）
  • 网络：可访问互联网（用于下载软件与安装插件）

三、实验过程

步骤一：安装 VS Code 并安装 Markdown 插件

从官网 https://code.visualstudio.com/ 下载安装包完成安装（勾选“添加到 PATH”）。启动后按 Ctrl + Shift + X 打开扩展面板，搜索并安装 Markdown All in One 插件，用于提供目录生成、快捷键、列表自动延续等写作增强功能。


图1：安装 Markdown All in One 插件（版本 3.6.3，作者 Yu Zhang）

步骤二：安装 Markdown Preview Enhanced 插件

继续在扩展面板中搜索并安装 Markdown Preview Enhanced 插件，该插件支持数学公式、Mermaid 流程图以及导出 PDF/HTML，用于增强 Markdown 的实时预览能力。


图2：安装 Markdown Preview Enhanced 插件（版本 0.8.30，作者 Yiyi Wang）

步骤三：创建并编辑 Markdown 文档，实时预览渲染效果

新建 test.md 文件，使用标题（#）、加粗（**）、无序列表（-）和代码块等基础语法编写内容。编辑时按 Ctrl + K V 开启并排预览，左侧源码与右侧渲染结果实时同步更新。文档源码如下：

五、知识总结

1. 代码编辑器 vs 集成开发环境（IDE）

类型 特点 典型代表
代码编辑器 轻量、启动快、以编辑为核心、靠插件扩展 VS Code、Sublime Text
IDE 功能全面，集成编译/调试/构建工具链 IntelliJ IDEA、PyCharm

2. 本实验涉及的常用插件

插件 主要功能
Markdown All in One 目录生成、快捷键、列表自动延续
Markdown Preview Enhanced 实时预览、数学公式、Mermaid、导出 PDF
Prettier 代码 / 文档格式化
markdownlint Markdown 语法与格式规范检查

3. Markdown 基础语法

语法 符号 作用
标题 # ~ ###### 一到六级标题
加粗 **文本** 文字加粗
无序列表 - / * 列表项
代码块 ``` 多行代码/语法高亮

4. 常用快捷键：Ctrl + Shift + X（扩展面板）、Ctrl + K V（并排预览）、Ctrl + ,（设置）。

六、出现问题

问题现象 原因分析 解决方法
扩展面板搜索不到插件 网络问题或关键词不准确 检查网络连接，使用插件全名精确搜索
预览窗口不随源码更新 未开启并排预览或焦点在预览侧 使用 Ctrl + K V 打开并排预览
代码块未高亮 未指定语言 在起始代码围栏后补上语言标识（如 javascript）

七、心得体会

通过本次实验，我掌握了 VS Code 的安装与插件扩展机制。在技术层面，我理解了代码编辑器“轻量内核 + 插件生态”的设计思想：编辑器本身只提供基础编辑能力，各种功能（Markdown 预览、格式化、语法检查）都通过插件按需叠加，这种模块化方式既保证了启动速度，又带来了极强的灵活性；同时也熟悉了 Markdown“用少量符号完成结构化排版”的简洁写法。

在思政层面，VS Code、Markdown All in One、Markdown Preview Enhanced 等工具均为开源免费软件，正是开源社区共享协作精神的体现——一个由个人开发者维护的插件能被近千万用户使用，说明开放共享能创造巨大价值。作为未来的软件工程师，我应善用并积极回馈开源生态，同时关注并支持国产编辑器与工具的发展，为自主可控的软件生态贡献力量。
