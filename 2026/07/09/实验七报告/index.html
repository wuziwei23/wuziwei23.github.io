实验七 远程登录管理 —— 实验报告

一、实验目的

安装并配置 OpenSSH 服务，实现多台主机之间的远程登录管理；配置 SSH 密钥认证实现免密登录；并掌握使用 SCP 与 rsync 进行跨机文件传输与同步的方法。

二、实验环境

  • 操作系统：Ubuntu 26.04 LTS，Linux 7.0.0-27-generic，x86_64
  • 主机：node1（192.168.56.101）、node2（192.168.56.102），用户名均为 my
  • 网络：两台虚拟机处于同一 Host-Only 网络（192.168.56.0/24）
  • 主要软件：OpenSSH Server、SCP、rsync

三、实验过程

步骤一：检查 SSH 服务状态

在 node2 上执行 systemctl status ssh，可见 ssh.service 处于 active (running) 状态，并已 enabled 开机自启，服务正在 0.0.0.0:22 与 :::22 端口监听。


图1：ssh.service 处于 active(running) 状态并监听 22 端口

步骤二：配置 SSH 服务器

用 sudo nano /etc/ssh/sshd_config 编辑配置文件，设置 PermitRootLogin no、PubkeyAuthentication yes 等关键项，保存后执行 sudo systemctl restart ssh 重启服务使配置生效。


图2：编辑 sshd_config 并重启 SSH 服务

步骤三：生成 SSH 密钥对

执行 ssh-keygen -t ed25519 -C "node1" 生成 ed25519 密钥对，密钥保存到 ~/.ssh/ 目录。ls ~/.ssh 可见私钥 id_ed25519 与公钥 id_ed25519.pub。


图3：生成 ed25519 密钥对及其 randomart 指纹

步骤四：将公钥复制到远程服务器

执行 ssh-copy-id my@192.168.56.102，首次连接确认主机指纹后输入一次密码，提示 Number of key(s) added: 1，公钥被写入远程主机的 authorized_keys。


图4：ssh-copy-id 成功安装公钥到远程主机

步骤五：测试免密登录

执行 ssh my@192.168.56.102，无需输入密码即直接登录到远程主机，显示 Ubuntu 欢迎信息，说明密钥认证配置成功。


图5：免密登录远程主机成功

步骤六：使用 SCP 传输文件

在 node1 上执行 scp ~/test.txt my@192.168.56.102:/home/my/ 将文件上传到 node2；登录 node2 后 ls 可见 test.txt，cat test.txt 输出 Hello Node2，确认文件传输成功。


图6：SCP 上传 test.txt 并在远程主机验证内容

步骤七：使用 rsync 同步文件

在 node1 上创建 ~/project/test.txt，执行 rsync -avz -e ssh ~/project/ my@node2:/home/my/project/ 将目录同步到 node2。输出显示 sending incremental file list 并成功传输 test.txt，实现了增量同步。


图7：rsync 增量同步 project 目录到远程主机

四、实验结果

测试项 命令 / 操作 结果

五、知识总结

1. SSH 认证方式对比

认证方式 原理 安全性 适用场景
密码认证 每次登录输入密码 较低 临时访问
密钥认证 公钥/私钥对验证 高 日常管理、自动化

2. 常用命令

命令 作用
systemctl status/restart ssh 查看 / 重启 SSH 服务
ssh-keygen -t ed25519 生成密钥对
ssh-copy-id 用户@主机 分发公钥实现免密
scp [-r] 源 目标 基于 SSH 复制文件/目录
rsync -avz -e ssh 源 目标 增量同步目录

3. SCP 与 rsync 对比：SCP 一次性整体复制，适合少量文件；rsync 只传输变化部分（增量同步），并可用 --delete、--exclude 等参数，适合目录同步与定期备份。

六、出现问题

问题现象 原因 解决方法
Permission denied 密码错误或密钥未配置 核对用户名密码，或用 ssh-copy-id 分发公钥
配置免密后仍提示输入密码 authorized_keys 权限不对 chmod 600 ~/.ssh/authorized_keys

七、心得体会

通过本次实验，我掌握了 SSH 远程登录与文件传输的完整流程，理解了密码认证与密钥认证的区别：密钥认证以公钥/私钥对替代明文密码，既免去了重复输入密码的麻烦，又显著提升了安全性，是多机管理和自动化运维的基础。SCP 与 rsync 则让跨机文件操作变得高效，rsync 的增量同步在实际备份场景中尤为实用。

在思政层面，OpenSSH 作为开源加密通信的代表，为全球服务器管理提供了免费而可靠的安全基础设施，体现了开源社区对信息安全的贡献。作为未来的软件工程师，熟练掌握安全的远程运维手段，既是基本功，也是对数据安全负责任的体现。
