实验六 Linux环境配置 —— 实验报告

一、实验目的

通过命令行完成 Linux 系统的基础操作与资源查看，了解常用系统配置文件（/etc/passwd、/etc/group 等）和用户配置文件（~/.bashrc），并掌握环境变量的查看、临时设置与永久设置方法。

二、实验环境

  • 操作系统：Ubuntu 24.04.4 LTS（Codename: noble）
  • 内核版本：Linux 6.17.0-35-generic，x86_64
  • 当前用户 / 主机：wuziwei / wuziwei（UID 1000）
  • 登录方式：SSH 远程登录（客户端 192.168.80.1）
  • Shell：Bash

三、实验过程

步骤一：查看系统信息

使用 whoami、uname -a、lsb_release -a、hostname、pwd 查看当前用户、内核架构、发行版版本、主机名和工作目录，确认系统为 Ubuntu 24.04.4 LTS，当前用户 wuziwei，主目录 /home/wuziwei。


图1：查看当前用户、内核与系统版本信息

步骤二：文件与目录基本操作

用 ls -lh 列出主目录内容，再依次执行 cd、mkdir、touch、cp、mv、rm 完成目录创建、文件创建、复制、移动与删除等常规操作。


图2：创建目录 test_dir、复制并移动 test.txt 等文件操作

步骤三：查看系统资源状态

用 top 查看进程与 CPU / 内存占用（内存共 3867.9 MiB），用 du -sh * 统计主目录下各子目录的大小。


图3：top 查看进程资源占用，du -sh 统计目录大小

步骤四：查看用户与组配置文件

用 cat /etc/group 查看用户组信息，用 id 查看当前用户的 UID、GID 及所属组，可见 wuziwei 属于 sudo、adm、users 等组，具备管理员权限。


图4：/etc/group 组信息与 id 命令输出

步骤七：查看环境变量

用 env 查看全部环境变量，再用 echo $PATH、echo $HOME、echo $USER 单独查看关键变量，可见 PATH 中已包含此前添加的 /home/wuziwei/myenv 路径。


图5：env 及 PATH / HOME / USER 变量查看

步骤八：临时设置环境变量

用 export 向 PATH 追加路径并创建变量 MY_VAR，echo 验证输出 Hello World，再用 unset 删除该变量。此类设置仅在当前终端会话有效。


图6：export 创建 / unset 删除临时环境变量

步骤九：永久设置环境变量（用户级）

用 nano ~/.bashrc 编辑用户配置文件，在末尾追加 PATH、自定义变量 MY_CUSTOM_VAR 和别名 ll，保存后执行 source ~/.bashrc 使其对当前用户生效。


图7：在 ~/.bashrc 末尾添加 PATH、自定义变量与别名

步骤十：永久设置环境变量（系统级 · 方法三）

用 sudo nano /etc/profile.d/myenv.sh 新建脚本并写入 export MY_GLOBAL_VAR="全局变量"，通过 /etc/profile.d/ 目录对所有用户生效，是推荐的系统级配置方式。


图8：在 /etc/profile.d/myenv.sh 中设置全局变量

五、知识总结

1. 常用命令

命令 作用
uname -a 查看内核与架构信息
lsb_release -a 查看发行版版本
id / groups 查看用户 ID 与所属组
env / echo $X 查看环境变量
export / unset 临时设置 / 删除变量
source 重新加载配置文件使其生效

2. 环境变量配置文件对比

配置文件 作用范围 加载时机 生效方式
~/.bashrc 当前用户 每次打开新终端 source ~/.bashrc
/etc/profile 所有用户 用户登录时 登录 / source
/etc/profile.d/*.sh 所有用户 用户登录时（推荐） 登录 / source
/etc/environment 所有用户 系统级，需重启 重启

3. 变量作用范围：export 直接设置的变量仅在当前会话有效；写入 ~/.bashrc 对当前用户永久生效；写入 /etc/profile.d/ 则对所有用户生效。

六、出现问题

问题现象 原因 解决方法
修改 ~/.bashrc 后不生效 未重新加载配置 执行 source ~/.bashrc
编辑 /etc/ 下文件被拒绝 普通用户权限不足 命令前加 sudo 提权

七、心得体会

通过本次实验，我熟悉了 Linux 命令行的基础操作，也理解了环境变量“临时”与“永久”、“用户级”与“系统级”之间的区别：export 只作用于当前会话，~/.bashrc 面向单个用户，而 /etc/profile.d/ 面向所有用户，配置的层级决定了它的作用范围。这种分层设计让系统在灵活性与统一管理之间取得了平衡。

在思政层面，Linux 作为开源操作系统的代表，其开放的源码和清晰的配置体系让每个使用者都能理解并掌控自己的系统，这正是开源精神的体现。作为未来的软件工程师，扎实掌握 Linux 环境配置是服务端开发与运维的基本功，也让我更加体会到自主可控的底层技术能力的重要性。
