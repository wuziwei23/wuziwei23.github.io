实验五 虚拟机安装与使用 —— 实验报告

一、实验目的

使用虚拟机工具 VirtualBox 安装 Linux 系统（Ubuntu），通过克隆技术构建一个多机虚拟化环境，并掌握基于虚拟机的开发环境的基本构建与网络配置方法（双网卡、静态 IP、主机名解析、多机互通）。

二、实验环境

  • 操作系统（宿主机）：Windows 11 家庭中文版（内部版本号 10.0.26100）
  • 虚拟化软件：Oracle VirtualBox 7.2.12
  • 客户机系统：Ubuntu 26.04（64 位）
  • 虚拟机配置：内存 2048 MB，处理器 2 核，硬盘 20 GB（VDI，动态分配）
  • 网络方案：双网卡（NAT + 仅主机 Host-Only）
  • 网络拓扑：Host-Only 网段 192.168.56.0/24，node1 = 192.168.56.101，node2 = 192.168.56.102
  • 网络：需要互联网连接

三、实验过程

步骤一：下载与安装 VirtualBox

访问 VirtualBox 官网 https://www.virtualbox.org/ 下载 Windows 版本安装包，双击启动安装向导并按默认选项完成安装，安装结束后重启电脑。


图1：Oracle VirtualBox 7.2.12 安装向导欢迎界面

步骤二：下载 Ubuntu ISO 镜像

访问 Ubuntu 官网 https://ubuntu.com/download 下载 Ubuntu 镜像（ubuntu-26.04-desktop-amd64.iso），供创建虚拟机时挂载使用。

步骤三：创建虚拟机（模板机）

在 VirtualBox 中新建虚拟机，类型选 Linux / Ubuntu (64-bit)，挂载下载好的 ISO 映像，分配内存 2048 MB、硬盘 20 GB（VDI 动态分配），并配置双网卡：适配器 1 为 NAT（负责上网），适配器 2 为仅主机（Host-Only，负责多机互联）。


图2：虚拟机 ubuntu-node1 的配置明细（内存 2048MB、处理器 2、硬盘 20GB、网卡 1 为 NAT）

步骤四：安装 Ubuntu

启动虚拟机进入 Ubuntu 安装向导，选择语言与键盘布局，网络保持默认 DHCP，存储选择使用整块磁盘，设置用户名和密码后等待系统复制文件完成安装并重启。


图3：Ubuntu 26.04 安装过程中，正在复制系统文件

步骤五：配置静态 IP 地址

登录系统后编辑 Netplan 配置文件，将 NAT 网卡（ens33）保持 DHCP，仅主机网卡（ens34）设为静态地址 192.168.56.101/24，再执行 sudo netplan apply 应用配置。通过 ip addr 可见两块网卡均已获得预期地址。


图4：应用 Netplan 后，ens33 为 NAT 网卡（192.168.8.136），ens34 为仅主机网卡（192.168.56.101）

步骤六：配置主机名与 hosts 解析

编辑 /etc/hosts，添加各节点的 IP 与主机名映射，使虚拟机之间可通过主机名互相访问。


图5：在 /etc/hosts 中添加 192.168.56.101 node1、192.168.56.102 node2 的映射

步骤七：克隆虚拟机

关闭模板机后，在 VirtualBox 中右键该虚拟机选择「克隆」，采用完全克隆方式克隆出 Ubuntu-node2（并重新生成网卡 MAC 地址）。


图6：克隆虚拟机向导，新虚拟机命名为 Ubuntu-node2

步骤八：修改克隆机的主机名和 IP

在克隆机上将主机名改为 node2，并把 Netplan 中的静态 IP 修改为 192.168.56.102 后应用。应用时提示配置文件权限过于开放，执行 sudo chmod 600 收紧权限后重新应用即可消除警告。


图7：node2 修改 IP 后应用配置，并用 chmod 600 消除权限过开警告

步骤九：验证多机环境

在 node1 上分别 ping 外网、node2 的私网地址与主机名，在 node2 上 ping node1，均连通且丢包率为 0%，说明双网卡与多机互通环境搭建成功。


图8：node1 上 ping 外网 8.8.8.8、192.168.56.102、node2 均成功，0% 丢包


图9：node2 上 ping node1（192.168.56.101）成功，0% 丢包

四、实验结果

各项验证结果汇总如下：

测试项 源 目标 结果
外网连通性 node1 8.8.8.8 ✅ 成功
多机 IP 连通性 node1 node2（192.168.56.102） ✅ 成功
主机名解析连通性 node1 node2 ✅ 成功
反向连通性 node2 node1（192.168.56.101） ✅ 成功

最终成果：成功安装 VirtualBox 并部署 Ubuntu 虚拟机，通过完全克隆构建出 node1、node2 两台虚拟机，配置双网卡与静态 IP 后实现了「虚拟机既能上网、又能内网互通」的多机虚拟化环境。

五、知识总结

1. 虚拟机网络三种模式

网络模式 特点 本实验用途
NAT 虚拟机借宿主机上网，外部不可直达 上网、下载软件
桥接（Bridged） 虚拟机直接接入宿主机所在局域网 本实验未使用
仅主机（Host-Only） 宿主机与虚拟机形成隔离私网 多机内网互通

2. 关键命令

命令 作用
sudo nano /etc/netplan/*.yaml 编辑网络（IP）配置
sudo netplan apply 应用 Netplan 网络配置
ip addr 查看网卡与 IP 地址
sudo hostnamectl set-hostname 修改主机名
sudo chmod 600 *.yaml 收紧配置文件权限
ping -c 3 <目标> 测试网络连通性

3. 双网卡方案要点：NAT 网卡（ens33）保持 DHCP 用于访问外网，仅主机网卡（ens34）配置同网段静态 IP 用于多机互联；/etc/hosts 提供主机名到 IP 的本地解析，克隆时应重新生成 MAC 地址以避免网卡冲突。

六、出现问题

问题现象 原因分析 解决方法
netplan apply 提示配置文件权限过开 Netplan 配置文件权限过大 执行 sudo chmod 600 /etc/netplan/*.yaml 后重试
克隆机网络异常 克隆保留了原网卡 MAC 地址 克隆时勾选重新生成 MAC 地址
虚拟机之间 ping 不通 Host-Only 网卡 IP 不在同一网段 检查各机 IP 是否均在 192.168.56.0/24 网段

七、心得体会

通过本次实验，我完成了从零搭建多机虚拟化环境的完整流程。在技术层面，我理解了虚拟机双网卡的分工思路：NAT 负责对外上网、仅主机负责内网互通，二者互不干扰；也体会到 Netplan 静态 IP 配置、/etc/hosts 主机名解析以及克隆技术的作用——克隆让「配好一台、复制多台」成为可能，是构建集群实验环境的高效手段。

在思政层面，VirtualBox 与 Ubuntu 都是自由/开源软件，让我在个人电脑上零成本模拟出多台服务器组成的实验环境，这正是开源生态带来的普惠价值。同时我也意识到，虚拟化与云计算的底层技术长期由国外主导，作为未来的软件工程师，应在善用开源工具的同时，关注并支持国产虚拟化与自主可控技术的发展。
